# 阿里云资源命名规范 — 对比报告

> **基准版本：** [Naming Convention (EAB 空间)](https://manulife-asia.atlassian.net/wiki/spaces/EAB/pages/473629055/Naming+Convention) — 2025-06-06
> 
> **对比版本：** [Ali Cloud - Resource (CGT 空间)](https://manulife-asia.atlassian.net/wiki/spaces/CGT/pages/1561199368/Ali+Cloud+-+Resource) — 2026-05-13
> 
> **报告日期：** 2026-06-12

---

## 📋 总结

### ✅ 保留未变
- 业务账户基本示例 (`MFC-AL-HKShareSvc-HKCOMMON-*`)
- ACK 和 API Gateway 的命名格式

### 🔄 主要修改（核心变化）
1. **资源组格式** — 新增 Pillar/vendor，删除 Product 段，长度限制信息丢失
2. **PaaS 基本格式** — 新增 Pillar 段
3. **PaaS 命名** — 从通用示例变为 MIL→MMS 真实迁移对比
4. **Appcode 具体化** — 从 `gcr/ods/reg` 等通用代码改为 `hnc-claims-core` 具体业务标识

### ⚠️ 完全删除（重要！）
1. **ECS 命名整节** — 包括 ECS、磁盘、NIC、Snapshot、EIP 所有子资源规则
2. **安全组命名整节** — 包括 VPC 关联规则
3. **网络命名整节** — 包括 17 个具体 VPC 列表、ALB/NLB/GWLB/NATGW/VBR 规则
4. **所有 Segment 详细表** — 长度限制、模板列表、选项定义等结构化信息丢失

### ➕ 完全新增
1. **MIL→MMS 迁移对比表** — 8 种 PaaS 服务的旧到新命名映射
2. **KMS 子资源命名** — Keys、Tags、Secret、RRSA 命名规则
3. **IDaaS 注册** — M2M Service Provider/Consumer 命名
4. **Namespace 映射** — Azure ↔ AliCloud 对应关系
5. **POT 账户类型** — 新增 GWLB、CloudPC 账户

| `MFC-AL-LMA-Prod` / `MFC-AL-LMA-NonProd` | Monitor |

## 3. ECS 命名 ⚠️ 整节删除

| 维度 | EAB (基准) | CGT (对比版) | 差异说明 |
|------|-----------|-------------|---------|
| **ECS 命名规则** | ✅ 详细 6 段表 (Hosting provider/OS/Server Type/Environment/Freeform/Sequence) | ❌ **完全删除** | ⚠️ **删除整节** |
| **ECS 相关资源** | ✅ 磁盘/NIC/Snapshot/EIP 命名 | ❌ **完全删除** | ⚠️ **删除整节** |

**EAB 原有 ECS 命名规则（已丢失）：**

- **格式：** `<HostingProvider><OSFamily><ServerType><Environment><Freeform><Sequence>`
  - Hosting: `AL=Alibaba Cloud`
  - OS: `W=Windows, L=Linux`
  - Server Type: `AP=Application, DB=Database, DC=Domain Controller`
  - Environment: `P=Production, Q=Production-Pre, N=NonProduction, U=UAT, D=Dev, S=SIT`
  - 示例：`ALLAPPETSABKP01`

- **ECS 相关资源（磁盘/NIC/Snapshot/EIP）：**
  - 格式：`<Type>-<Environment>-<Region>-<ECSName>-<Sequence>`
  - 示例：`dsk-prd-hkg-ALLAPPETSABKP01-01`
  - 类型：`dsk / snp / eni / eip`

## 6. PaaS 服务命名

| 维度 | EAB (基准) | CGT (对比版) | 差异说明 |
|------|-----------|-------------|---------|
| **基本格式** | `<Freeform>-MFC-<Cloud Service>-<env>-<region>-<appcode>-<seq>` | `<Freeform>-MFC-<Cloud Service>-<env>-<region>-<Pillar>-<Appcode>-<seq>` | 🔄 **修改** — 新增 Pillar |
| **PaaS Segment 表** | ✅ 完整长度/模板表 | ❌ **删除** | ⚠️ **删除** |
| **服务类型对比** | ❌ 仅列出名称 | ✅ **MIL vs MMS** 双列对比 | ➕ **重大新增** |

### PaaS 服务详细对比 (MIL → MMS 变化)

| 服务类型 | MIL (旧) | MMS (新) | 差异 |
|---------|---------|---------|------|
| **ACK** | `mfc-ack-dev-hkg-gcr-01` 等 | `mfc-ack-dev-hkg-gcr-01` 等 | ➖ 相同，无变化 |
| **API Gateway** | `mfc-apigw-nprod-hkg-dev-01` | `mfc-apigw-nprod-hkg-dev-01` | ➖ 相同，无变化 |
| **KMS** | 简单格式 | **大幅扩展**：增加 Keys、KMS Secret、RRSA 子资源命名 | ➕ **显著新增** |
| **Simple Log Service** | `k8s-applog-mfc-ack-dev-hkg-gcr-01` | `k8s-applog-mfc-ack-dev-hkg-hnc-claims-core-01` | 🔄 Appcode 从 `gcr` 改为具体应用 |
| **PolarDB** | `mfc-polarpgsql-dev-hkg-hnc-claims-ods-01` | `mfc-polarpgsql-dev-hkg-hnc-claims-core-01` | 🔄 从 `ods` 改为 `core`，附定价/多环境 follow-up |
| **Tair (Redis)** | `NA` (未使用) | `mfc-redis-dev-hkg-hnc-claims-core-01` 等 | ➕ **新增** |
| **OSS** | `NA` (未使用) | `mfc-oss-dev-hkg-hnc-claims-core-01` 等 | ➕ **新增** |
| **RocketMQ** | `NA` (未使用) | `mfc-rocketmq-dev-hkg-hnc-claims-core-01` 等 | ➕ **新增** |

### KMS 子资源命名 (CGT 完全新增)

| 子资源 | 命名示例 |
|-------|---------|
| Keys | `key_nprd_hkg_hnc_claims_core_01` |
| Tags | `hnc_claims_core_kms_non_prod` |
| KMS Secret | `secret_dev_hkg_hnc_claims_core_01` |
| RRSA | `rrsa-dev-gcr-hkg-hnc-claims-core-01` |

### PolarDB 待跟进事项 (CGT 新增)
1. 不同 DB 之间如何不影响
   - 不同实例按用途划分：domain instance / application instance / vendor instance
2. 定价模型：pay as you go
   - 无需单独为性能测试分离 DB 实例
   - DB 实例配置默认值的成本 → 需确认 Ali 团队
3. 多环境
   - 同一实例下多个数据库：`ichs_01, ichs_02, ichs_03, ichs_04`

## 🎯 风险评估与建议

**风险：** CGT 版本更关注上层 PaaS 和业务迁移需求，但**缺失了大量底层基础设施（ECS/网络/安全组）的命名规范**。

**建议：**
1. **立即补充** — 将 EAB 的 ECS、网络、安全组命名规则补充进 CGT 文档
2. **统一来源** — 保留 CGT 版本作为主文档，但恢复所有 Segment 详细表（长度限制、选项列表等）
3. **补充资源组 Product 段** — CGT 删除了 Product 选项，建议恢复
4. **EAB 原文作为备份参考** — 保留 EAB 文档链接，以防后续需要查询原始定义
