# Carlos Sanchez

**Role:** Principal Scientist, [[3 - Areas/AIOps/wiki/entities/Adobe]]
**Affiliation:** Adobe
**Active in:** Open-source cloud-native tooling, progressive delivery, Kubernetes, CI/CD

---

## Relevance

Carlos Sanchez is the primary creator of the [[3 - Areas/AIOps/wiki/entities/Argo Rollouts]] AI analysis plugin demonstrated at KubeCon + CloudNativeCon. He combines Adobe's enterprise Kubernetes operations experience with open-source community involvement to build agentic tooling for [[3 - Areas/AIOps/wiki/concepts/Progressive Delivery|progressive delivery]] workflows.

---

## Contributions

- Created the AI canary analysis plugin for [[3 - Areas/AIOps/wiki/entities/Argo Rollouts]] (hosted in Argo product labs; actively developed)
- Designed the Java/Quarkus-based Kubernetes agent that performs LLM-powered log and cluster-state analysis during canary rollouts
- Demonstrated [[3 - Areas/AIOps/wiki/concepts/Self-Healing Systems|self-healing rollout]] architecture: detect → diagnose → rollback → GitHub issue → coding AI fix → re-deploy

---

## Affiliation to Other Adobe Work

Carlos Sanchez's work sits on the **release engineering** side of Adobe's cloud operations, complementing the **on-call incident response** agent work described by [[3 - Areas/AIOps/wiki/entities/Danilo Banjac]] and [[3 - Areas/AIOps/wiki/entities/Iveri Prangishvili]]. Together, they illustrate that Adobe's agentic AI investment spans the full software lifecycle — from production incidents to rollout failures.
